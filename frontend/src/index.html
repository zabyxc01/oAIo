QUICK START
1.Pick a mode in the LIVE tab — oLLMo (LLM), oAudio (voice), or comfyui-flex (image gen).
2.Services auto-start. Watch the telemetry strip for VRAM/RAM usage.
3.Use the service UIs — Open-WebUI (:3000) for chat, ComfyUI (:8188) for images.
4.Monitor everything in the LIVE tab — benchmark, API traffic, enforcement, kill log.
5.Customize nodes, routing, and resources in CONFIG.
If something goes wrong, hit the red KILL ALL button in the top bar to stop all services and clear modes instantly.

KEY CONCEPTS
MODES
Groups of services with VRAM budgets. Only one mode per GPU — activating a new mode displaces the old one. Three built-in: oLLMo (11 GB), oAudio (5 GB), comfyui-flex (18 GB).
ENFORCEMENT
Automatic OOM protection. Warns at 85% VRAM, hard-kills lowest-priority services at 95%. Configurable thresholds and per-service priority (1–5). Toggle on/off in LIVE.
SYMLINKS
The /mnt/oaio/ abstraction layer — 16 symlinks that decouple services from physical disk paths. Enables storage tiering (NVMe / SATA / RAM) without reconfiguring containers.
SERVICES
Docker containers managed by oAIo. Each has a group (oLLM, oAudio, Render, Control), priority, memory mode, and kill limit (soft warning vs hard cutoff).

TIPS
KILL ALL — Stops every running service and clears all active modes. Use when VRAM is stuck or a service is unresponsive.
Mode switching — Activating a new mode gracefully stops displaced services before starting new ones. No need to manually kill first.
Node editor — In CONFIG, drag to pan, scroll to zoom. Click a node to configure its resources. Double-click to open its service page.
VRAM ceiling — Set a virtual VRAM limit in the Enforcement card to test mode fits without real hardware constraints.
Telemetry strip — Toggle with the TELEMETRY pill in the top bar. Shows live VRAM, RAM, GPU utilization, and active modes.
Workflows — Save complete workspace state (modes, layout, routing, settings) in SETTINGS > Workflow Saves.
